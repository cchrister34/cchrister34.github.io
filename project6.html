About the project

This project was created as part of my games programming module during my third year at Northumbria University. The project involved creating a multiplayer third-person shooter game using Unreal Engine 5, with a focus on implementing core gameplay mechanics such as shooting, health, and player movement. The game features replicated gameplay elements, with the primary goal of creating a functional multiplayer experience that allows players to work together to survive against AI-controlled enemies.

What I learned

I learned fundamentals of multiplayer in Unreal Engine using the client-server model. Developing an understaning of replication across the network using Unreal's frameworks. I learned the importance of not overloading the server with packets to prevent lag and desync alongside how to send information across the network to maintain a stable connection while remaining synchronised.

Problems Faced

One of the key challenges we faced during this project was the learning process of understaning when to send RPCs to update client-side gameplay. This added layer of complexity proved to be initially challenging as getting the logic to work on the server was straightforward but, replicating the same logic to clients required knowledge of how Unreal Engine manages replication. But the more I faced this challenge, the more I understood when a function or variable should be replicated to ensure syncrhonised gameplay between clients.



Conclusion

Overall, this project was a valuable learning experience that helped me develop my skills in programming for multiplayer in Unreal Engine as well as refining my C++ skills. I now have knowledge of the fundamentals of replication in UE5 while maintaining clean networking to ensure good server performance.
